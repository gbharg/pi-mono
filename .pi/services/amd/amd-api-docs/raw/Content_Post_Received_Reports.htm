Post Received Reports

Overview

The POST /receivedreports endpoint allows the retrieval of Electronic Remittance Advice (ERA) reports.

Endpoint Details

  • URL: {{Redirect_URL_REST_PM}}/era/receivedreports

  • Method: POST

  • Content-Type: application/json

  • Authentication: Required (401 Unauthorized if not authenticated)

Request Body
Copy 
{
  "beginDate": "2026-01-01T00:00:00",
  "endDate": "2026-01-20T00:00:00",
  "archive": false,
  "title": "string",
  "reportText": "string"
}

Request Parameters

Parameter Type Required Description Validation Rules

beginDate

DateTime

Yes

Start date of the search range.

Must be between 1/1/1900 and 1/1/3000. Must be a date only (MM/dd/yyyy) without a time component. Cannot be after endDate.

endDate

DateTime

Yes

End date of the search range.

Must be between 1/1/1900 and 1/1/3000. Must be a date only (MM/dd/yyyy) without a time component.

archive

Boolean

No

Filter by archived status. Indicates whether to search for archived reports. Defaults to false.

-

title

String

No

Optional filter for the report title.

-

reportText

String

No

Optional text search within the report content. If provided, performs a full-text search within the report

-

Request Validation

The endpoint validates the following conditions:

  • beginDate cannot be after endDate

  • beginDate must not contain a time component

  • endDate must not contain a time component

  • Date values must be within the valid range (1/1/1900 - 1/1/3000)

Request Examples

Basic Search (Date Range and Archive Status)
Copy 
{
  "beginDate": "2026-01-01T00:00:00",
  "endDate": "2026-01-20T00:00:00",
  "archive": false
}
Search with Title Filter
Copy 
{
  "beginDate": "2026-01-01T00:00:00",
  "endDate": "2026-01-20T00:00:00",
  "archive": false,
  "title": "Payor Claim Data"
}
Search with Report Text (Full-Text Search)
Copy 
{
  "beginDate": "2026-01-01T00:00:00",
  "endDate": "2026-01-20T00:00:00",
  "archive": false,
  "title": "Payor Claim Data",
  "reportText": "patient123"
}

Response Schema

Field Type Description

queryTimeSeconds

String

Execution time of the query in seconds (formatted to 3 decimal places)

reportsCount

Integer

Number of reports returned in the reports array

reports

Array

Collection of received reports matching the search criteria

Report Object Schema

Each report object in the reports array contains:

Field Type Description

read

String

Has the report been read? ("1" or "0")

parsed

String

Has the report been parsed for the claim lifecycle? ("1" or "0")

dateReceived

String

Date the report was received (formatted as MM/dd/yyyy)

dateReceivedSortable

Integer

Sortable version of the date received (yyyyMMdd format)

title

String

Title of the report

id

Integer

Unique identifier for the report

claimFID

Integer

Obsolete field, the value is always null

archived

String

Has the report been archived? ("1" or "0")

display

String

Is the report still active/not deleted? ("1" or "0")

Error Responses

400 Bad Request – Returned when request validation fails.

Common Validation Errors:

  • beginDate is after endDate

  • Date values contain time components

  • Date values are outside the valid range

  • Required fields are missing

Implementation Notes

Search Behavior

  • When reportText is not provided: The API searches using beginDate, endDate, archive, and title parameters only.

  • When reportText is provided: The API performs a more comprehensive search, including full-text search within the report content. This may result in slower query execution but provides more precise results.
