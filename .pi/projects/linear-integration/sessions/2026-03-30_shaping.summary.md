## Recap
Started the day deploying the compaction extension built overnight. Then set up the Linear app user — discovered the delegation model (delegateId vs assigneeId, AgentSessionEvent webhooks). That led into a deep shaping session on how the entire orchestration system should work.

We went through it methodically: entry points (iMessage synchronous, Linear async), task management (everything is a Linear issue, Gautam adds, I manage), workflow states (two levels: project and issue, with Plan gating execution), git model (always branch, no project branches, every change PRs to main), sub-agent lifecycle (kill after In Review, sessions saved in git notes + Linear), review workflow (two gates: reviewer agent + 4-agent PR review), and cloud environments (default for code changes, separate project to scope).

Key breakthrough: Linear becomes the orchestration bus. I delegate to any agent (local sub-agents, Claude on MBP, Codex, Gemini) by assigning Linear issues. Each agent streams progress via AgentActivity. Git Notes preserve full session logs on commits. The system scales across machines without Pi being a bottleneck.

Gautam pushed back on several assumptions: project branches aren't needed (continuous integration is better), specs and plans are the same thing, cloud envs might be needed for every sub-issue (still discussing), and sub-agent roles should come from actual usage data not theory.
